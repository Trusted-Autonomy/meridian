# Example: `meridian setup`

Fictitious company: **Stackwise Inc.** — B2B SaaS developer tools (25 people, Series A).
Their product is an engineering metrics dashboard that connects GitHub, Linear, and Slack.
They're in the middle of an enterprise sales motion and need SOC 2 by Q4.

## What was written

```toml
# meridian.toml — generated by meridian setup

[source]
type = "claude-code"

[report]
format = "table"

[[kpi]]
id = "customer_value"
label = "Customer Value"
description = "Features that directly advance customer outcomes and retention"
weight = 1.2

[[kpi]]
id = "engineering_health"
label = "Engineering Health"
description = "Code quality, velocity, and system reliability"
weight = 1.0

[[kpi]]
id = "security_trust"
label = "Security & Trust"
description = "Security posture, compliance, and data integrity"
weight = 1.1

[[kpi]]
id = "revenue_growth"
label = "Revenue Growth"
description = "Activities that advance pipeline and conversion"
weight = 1.3
```

At this point Stackwise has a working config. They run `meridian report` immediately
and get useful output — KPIs are scored by keyword alignment. For richer KPIs with
metrics, target values, and a red-team review, they'd run `meridian pm init` instead
(see `pm-init.md`).
